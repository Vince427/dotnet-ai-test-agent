Portable desktop UI testing

AgentLoop Desktop Test Agent

A manual-first runner for existing .NET desktop apps. YAML plans, CLI commands, readable artifacts, and the static AgentLoop Workbench are the product contract.

YAML specs -> CLI runner -> Artifacts -> AgentLoop Workbench

Quickstart

  1. Edit or add a YAML business test under tests/.
  2. Validate plans without .env or LLM access.
  3. List tests, then run one selected test against a launched desktop app.
  4. Review summary.md, report.json, screenshots, and the AgentLoop Workbench.

Validate and List

dotnet run --project .\src\AgentRunner\AgentRunner.csproj -f net8.0-windows -- --validate-plan --format json

dotnet run --project .\src\AgentRunner\AgentRunner.csproj -f net8.0-windows -- --list-tests --format json

Run and Inspect

dotnet run --project .\src\AgentRunner\AgentRunner.csproj -f net8.0-windows -- --suite smoke --test-id LOGIN-001 --evidence-level standard

powershell -ExecutionPolicy Bypass -File .\scripts\render-ui.ps1

WinForms and WPF Examples

Focused example plans live under tests/examples/winforms/ and tests/examples/wpf/. They cover login, profile save, radio/combo/list/grid controls, modal confirmation, disabled states, and async loading before MAUI and Avalonia parity. 

powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\run-ui-examples.ps1 -WhatIf

powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\run-ui-examples.ps1 -Framework winforms -Scenario login
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\run-ui-examples.ps1 -Framework wpf -Scenario controls

Boundaries

  • AgentLoop tests existing apps from the outside; target apps do not need agent-specific packages or production code.
  • OpenRouter and other LLM providers are optional runtime assistance, not prerequisites for validation, listing, docs, or workbench rendering.
  • Some generated files still use symphony while the AgentLoop rename finishes.
